# Silver Care IoT 进度记录

**日期：** 2026-06-01

---

## Done

### 后端（已有，本次未改动）
- TCP 设备网关（端口 9001），接收手表协议报文
- 协议解析：LK 心跳、UD/UD2 位置、btemp2 体温、bphrt 血压心率、AL 报警
- 设备在线/离线状态管理
- 原始报文日志存储
- Admin REST API（`/api/admin/**`，Basic Auth）
  - 设备列表、设备详情
  - 最新健康数据、健康记录列表
  - 最新位置、位置记录列表
  - 原始报文分页查询
  - 发送指令到设备

### 后端（本次新增）
- MiniApp API（`/api/miniapp/**`，无需认证）
  - `GET /api/miniapp/devices/{deviceNo}/overview` — 设备状态 + 最新健康 + 最新位置
  - `GET /api/miniapp/devices/{deviceNo}/health-records?size=20` — 健康记录列表
  - `GET /api/miniapp/devices/{deviceNo}/location-records?size=20` — 有效位置记录列表
- `MiniappSecurityConfig` — 放行 miniapp 路径，不影响 admin 认证
- 修复 `LocationDataServiceTest` 时区 bug（ZoneId.systemDefault → ZoneOffset.UTC）

### Vue3 Web 管理后台（本次新增，`web/`）
- 技术栈：Vue 3 + Vite + Element Plus + Vue Router + Axios
- 设备列表页：分页表格，在线/离线状态标签
- 设备详情页：基本信息、健康记录表格、位置轨迹（腾讯地图 polyline）、发送指令
- 原始报文页：分页表格，解析状态标签
- `LocationMap.vue` 组件：腾讯地图 GL，渲染最新位置 marker + 历史轨迹线
- Axios 实例统一配置 Basic Auth（从 `.env.local` 读取凭证）

### 微信小程序（本次新增，`miniapp/`）
- 技术栈：原生微信小程序（WXML + WXSS + JS）
- 首页：输入设备编号查询，缓存最近 5 条记录
- 概览页：设备状态、电量、步数、最新健康数据、最新位置
- 健康数据页：最新数据卡片 + 历史记录列表（最近 20 条）
- 位置轨迹页：原生 `<map>` 组件，polyline 轨迹 + 位置记录列表
