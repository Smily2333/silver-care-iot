## 一、通用数据格式
所有通信数据统一遵循以下格式：
```
[厂商标识*设备ID*内容长度*内容]
```

- **厂商标识**：固定为2个字节，本协议统一使用 `3G`
- **设备ID**：终端唯一标识（字符串格式）
- **内容长度（LEN）**：
  - 固定为4个字节的十六进制ASCII码，**高位在前，低位在后**
  - 示例：`00A2` 表示十进制162，`000D` 表示十进制13
  - 长度仅计算**内容部分**的字节数，不包含外层方括号 `[]` 和分隔符 `*`
  - 逗号分隔符 `,` 计入内容长度
  - 中文/Unicode字符按实际字节数计算
- **内容**：指令+参数，参数之间用英文逗号 `,` 分隔

## 二、终端发送指令
### 1. 链路保持
- 终端发送：
  ```
  [3G*设备ID*LEN*LK,步数,翻滚次数,电量百分数]
  ```
  实例：`[3G*8800000015*000D*LK,50,100,100]`
- 平台回复：
  ```
  [3G*设备ID*LEN*LK]
  ```
  实例：`[3G*8800000015*0002*LK]`
- 说明：终端按设定间隔发送，若未收到平台回复则重新建立连接

- 终端发送：
  ```
  [3G*设备ID*LEN*TK,ARM格式音频数据]
  ```
- 平台回复：

- 终端发送：
  ```
  [3G*设备ID*LEN*PP,手表当前时间,位置数据]
  ```
  实例：
  ```
  [3G*001*000D*PP,091046,1916,3,22.50193,N,113.61950,E,0.0,60.3,0.0,9,100,100,0,0,00000010,7,255,460,1,9529,21809,160,9529,21405,133,9529,63555,133,9529,63554,124,9529,21242,119,9529,21151,118,9529,63574,116,0,23.2]
  ```
- 平台回复：
  - 成功：`[3G*设备ID*LEN*pp,对方设备ID]`
  - 失败：`[3G*设备ID*LEN*pp,错误码]`
    - 错误码`1`：对方已交过好友
    - 错误码`2`：自己已交过好友

### 8. 体温上报（新版 BT=2）
- 终端发送：
  ```
  [3G*设备ID*LEN*btemp2,测量模式,体温值]
  ```
  - 测量模式：`0`=额头模式，`1`=手腕模式
  - 体温值：`0`=温度过低异常，`1`=温度过高异常，其他为实际体温（摄氏度浮点数）
- 平台回复：`[3G*设备ID*LEN*btemp2]`

### 9. 血压心率上传
- 终端发送：
  ```
  [3G*设备ID*LEN*bphrt,高压,低压,心率,身高,性别,年龄,体重]
  ```
  - 高压/低压/心率：`0`表示无效
  - 性别：`1`=男，`2`=女
  - 身高：单位厘米；体重：单位千克
- 平台回复：`[3G*设备ID*LEN*bphrt]`

### 10. 远程拍照图片上传
- 终端回复：
  ```
  [3G*设备ID*len*img,时间戳,y,z]
  ```
  - 时间戳：秒级时间戳
  - z：照片二进制内容

  ```

  ```
  实例：`[3G*9403043989*009F*WT,2016-09-26,11:00:00,晴,0,22,12,26,深圳]`
- 说明：
  - 天气描述：GB2312编码

- 平台发送：`[3G*设备ID*len*APPLOCK,WX-1,PH-1,DW-1,YJ-1]`
- 开关说明：
  - PH：手机模式（默认开）
  - DW：GPS定位（默认开）
  - YJ：夜间模式（默认关）
- 终端回复：`[3G*设备ID*len*APPLOCK]`

### （二）老人特殊功能协议
#### 1. 心率协议
- 平台发送：`[3G*设备ID*LEN*hrtstart,x]`
- 参数x：
  - `0`：关闭心率上传
  - `1`：单次心率上传
  - `300-65535`：连续上传间隔（秒）
- 实例：`[3G*8800000015*LEN*hrtstart,300]`
- 终端回复：`[3G*设备ID*LEN*hrtstart]`

#### 2. 跌倒报警灵敏度设置（LS=3+6）
- 平台发送：`[3G*设备ID*LEN*LSSET,当前值+最大值]`
- 实例：`[3G*设备ID*LEN*LSSET,3+6]`
- 说明：数值越大，灵敏度越低
- 终端回复：`[3G*设备ID*LEN*LSSET]`

#### 3. 跌倒报警协议
- 平台发送：`[3G*设备ID*LEN*FALLDOWN,X,Y]`
- 参数：
  - X：跌倒报警开关（0=关，1=开）
  - Y：跌倒后是否拨打中心号码（0=否，1=是）
- 终端回复：`[3G*设备ID*LEN*FALLDOWN]`

### （三）3G/4G特殊协议

### 附录三：终端状态位详细定义（32位十六进制）
终端状态字段为**8位十六进制字符串**（对应32位二进制），格式为：`00000000`
- 高16位（前4位十六进制）：**报警状态位**，1表示对应报警触发
- 低16位（后4位十六进制）：**设备状态位**，1表示对应状态生效
- 位编号：从右往左（最低位）为第0位，最高位为第31位

| 位编号 | 十六进制掩码 | 含义               | 说明                                   |
|--------|--------------|--------------------|----------------------------------------|
| **低16位（设备状态位，Bit0-Bit15）** |
| 0      | 00000001     | 预留               | 未使用                                 |
| 1      | 00000002     | 手表静止状态       | 1=静止，0=移动                         |
| 2-3    | 0000000C     | 预留               | 未使用                                 |
| 4      | 00000010     | 手环佩戴状态       | 1=已取下，0=已佩戴                     |
| 5-15   | 0000FFE0     | 预留               | 未使用                                 |
| **高16位（报警状态位，Bit16-Bit31）** |
| 16     | 00010000     | 进围栏状态         | 1=进入电子围栏                         |
| 17     | 00020000     | 低电报警           | 1=电量低于阈值                         |
| 18     | 00040000     | SOS报警            | 1=手动触发SOS                          |
| 19     | 00080000     | 出围栏报警         | 1=离开电子围栏                         |
| 20     | 00100000     | 跌倒报警           | 1=检测到跌倒                           |
| 21     | 00200000     | 心率异常报警       | 1=心率超出正常范围                     |
| 22     | 00400000     | 取下手环报警       | 1=检测到手环被取下                     |
| 23-31  | FF800000     | 预留               | 未使用                                 |

> ⚠️ **原文档笔误修正**：原文档中"第4位为1表示手表静止"为错误描述，实际为**Bit1（掩码00000002）**。
> 验证：协议实例中状态值`00000010`对应二进制`00000000 00000000 00000000 00010000`，Bit4为1，表示**手环已取下**。
